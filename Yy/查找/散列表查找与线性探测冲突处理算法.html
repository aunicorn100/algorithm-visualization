例题9.8：散列表(Hash Table)查找与冲突处理详解

演示散列表的构建、查找过程，以及如何处理哈希冲突（以开放定址法为例）

散列表的基本概念与冲突处理

哈希函数 (Hash Function)

将关键字 key 映射到散列表中存储位置的函数，记为 H(key)。一个好的哈希函数应尽量减少冲突。

常用方法：直接定址法、数字分析法、平方取中法、折叠法、除留余数法（H(key) = key % p）。

哈希冲突 (Hash Collision)

不同的关键字通过同一个哈希函数得到相同的哈希地址，即 H(key1) = H(key2) 且 key1 ≠ key2。

冲突不可避免，只能尽量减少并妥善处理。

开放定址法 (Open Addressing)

当冲突发生时，按照某种规则在散列表中寻找下一个空的存储位置。

常用探测序列：线性探测（d_i = i）、二次探测（d_i = i²）、伪随机探测。

链地址法 (Separate Chaining)

将所有哈希地址相同的记录链接在同一个单链表中。散列表的每个单元存储一个指针，指向对应链表的头节点。

优点：处理冲突简单，无堆积现象，平均查找长度较短。

// HashSearch: 在散列表 HT 中查找关键字 key
// 散列表大小为 m, 哈希函数 H(key) = key % p
int HashSearch(int HT[], int m, int key, int p) {
    int addr = H(key, p); // 计算初始哈希地址
    int i = 1;
    // 线性探测，直到找到空位置或目标关键字
    while (HT[addr] != NULLKEY && HT[addr] != key) {
        addr = (addr + 1) % m; // 线性探测下一个地址
        i++;
        if (i > m) { // 探测次数超过表长，说明表满或关键字不存在
            return -1;
        }
    }
    if (HT[addr] == key) {
        return addr; // 查找成功，返回存储位置
    } else {
        return -1; // 查找失败
    }
}

// 哈希函数: 除留余数法
int H(int key, int p) {
    return key % p;
}
