例3.8 循环单链表链队演示

算法概述

例3.8采用不带头结点只有一个尾结点指针rear的循环单链表存储队列，通过尾结点指针rear唯一标识整个队列。该结构的核心特点是：

  • 队空条件：rear == NULL
  • 队满条件：不考虑（链表存储，理论上无容量限制）
  • 进队操作：将新结点插入到循环单链表的表尾
  • 出队操作：删除循环单链表的首结点（队头）

关键原理

循环单链表的尾结点next指针指向首结点，形成闭合环。通过尾指针rear可直接访问尾结点，同时通过rear->next访问首结点（队头），实现高效的进队和出队操作。

算法代码与解析 

// 循环单链表节点类型
typedef struct LNode {
    ElemType data;          // 数据元素
    struct LNode *next;     // 指向后继节点的指针
} LinkList;

// 队列通过尾指针rear唯一标识
// 无需额外的队列结构体，直接使用LinkList*作为尾指针

代码解析：

  • 采用不带头结点的循环单链表存储队列元素
  • 每个节点包含数据域data和指针域next
  • 队列仅通过尾指针rear标识，无需额外的队列结构体
  • 尾指针rear指向链表的最后一个节点（队尾）
// 初始化队列算法
void InitQueue(LinkList *&rear) {
    rear = NULL;  // 队空时，尾指针置为NULL
}

代码解析：

  • 初始化操作非常简单，只需将尾指针rear设置为NULL
  • 此时队列为空，没有任何节点存在
  • 时间复杂度：O(1)，仅需一次指针赋值操作
// 进队算法
bool enQueue(LinkList *&rear, ElemType x) {
    LinkList *p;
    // 创建新节点
    p = (LinkList *)malloc(sizeof(LinkList));
    p->data = x;
    
    if (rear == NULL) {  // 原队列为空
        p->next = p;     // 新节点的next指向自身，形成循环
        rear = p;        // rear指向新节点（既是队头也是队尾）
    } else {             // 原队列非空
        p->next = rear->next;  // 新节点的next指向原队头
        rear->next = p;        // 原队尾的next指向新节点
        rear = p;              // rear指向新节点（新队尾）
    }
    return true;
}

代码解析：

  • 进队操作分两种情况：队空和队非空
  • 队空时：新节点的next指向自身，形成循环，rear指向该节点
  • 队非空时：新节点插入到队尾，rear更新为新节点
  • 核心操作：维护循环链表的闭合性，确保rear->next始终指向队头
  • 时间复杂度：O(1)，仅需几次指针操作，无需遍历链表
// 出队算法
bool deQueue(LinkList *&rear, ElemType &x) {
    LinkList *q;
    
    if (rear == NULL) {  // 队空，出队失败
        return false;
    } else if (rear->next == rear) {  // 队列只有一个节点
        x = rear->data;  // 取出节点数据
        free(rear);      // 释放节点空间
        rear = NULL;     // 队空，rear置为NULL
    } else {             // 队列有两个或以上节点
        q = rear->next;  // q指向队头节点
        x = q->data;     // 取出队头节点数据
        rear->next = q->next;  // 队尾的next指向新队头
        free(q);         // 释放原队头节点空间
    }
    return true;
}

代码解析：

  • 出队操作分三种情况：队空、队列仅一个节点、队列多个节点
  • 队空时：直接返回false，出队失败
  • 仅一个节点时：取出数据后释放节点，rear置为NULL
  • 多个节点时：删除队头节点，更新rear->next指向新队头
  • 核心操作：通过rear->next找到队头，维护循环链表结构
  • 时间复杂度：O(1)，无需遍历链表，仅需几次指针操作
// 判空算法
bool queueEmpty(LinkList *rear) {
    return (rear == NULL);  // rear为NULL表示队空
}

代码解析：

  • 判空逻辑非常简单，只需检查尾指针rear是否为NULL
  • 返回true表示队空，false表示队列非空
  • 时间复杂度：O(1)，仅需一次指针比较操作
  • 该判空逻辑贯穿整个算法，在进队、出队前都需要判断队列状态

初始化队列（InitQueue）

初始化操作将队列的尾指针rear设置为NULL，此时队列为空，没有任何节点存在。

操作效果：

  • rear = NULL
  • 队列长度 = 0
  • 无任何节点，队列为空

第一个元素进队（enQueue）

当队列为空时，进队操作会创建新节点，新节点的next指针指向自身（形成循环），尾指针rear指向该节点。

核心操作：

  1. 创建新节点p，p->data = 元素值
  2. p->next = p（节点自循环）
  3. rear = p（rear指向新节点）

多个元素进队（enQueue）

当队列非空时，新节点插入到队尾，新节点的next指向原队头，原队尾的next指向新节点，rear更新为新节点。

核心操作（以插入B、C为例）：

  1. 创建节点B，B->next = rear->next（指向A）
  2. rear->next = B（A->next = B）
  3. rear = B（rear指向B）
  4. 重复上述步骤插入节点C

单个元素出队（deQueue）

出队操作删除队头节点（rear->next指向的节点），更新rear->next指向新队头，释放原队头节点空间。

核心操作（删除A为例）：

  1. q = rear->next（q指向队头A）
  2. x = q->data（取出A的数据）
  3. rear->next = q->next（C->next = B）
  4. free(q)（释放A节点）

全部元素出队（deQueue）

依次删除队列中所有元素，直到队空。当队列中只剩一个节点时，删除后将rear置为NULL，恢复初始空状态。

核心操作（删除B→删除C）：

  1. 第一次出队：删除队头B，队列剩余C（C->next=C）
  2. 第二次出队：删除最后一个节点C，rear置为NULL
  3. 最终状态：队空，无任何节点
